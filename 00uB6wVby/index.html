群晖创建macvlan，与宿主机容器通讯

发布于 2023-03-02 | 分类于 群晖 、 macvlan 、 docker | 2分钟 | 239字数 | 浏览量：

默认开启macvlan之后无法与宿主机容器通信。需要单独配置。

群晖修改网卡设置

勾选下图

创建macvlan

docker network create -d macvlan --subnet=10.10.10.0/24 --gateway=10.10.10.254 --ipv6 --subnet=fe80::/64 --gateway=fe80::1 -o parent=ovs_eth0 ttt

修改网段，网关，开启了ipv6

创建群晖开机脚本

vim /usr/local/lib/systemd/system/macvlan.service

配置启动和停止脚本

vim /usr/local/bin/macvlan_start.sh

ip link add ggg link ovs_eth0 type macvlan mode bridge
ip addr add 10.10.10.99 dev ggg
ip link set ggg up
ip route add 10.10.10.100 dev ggg
ip route add 10.10.10.101 dev ggg
ip route add 10.10.10.102 dev ggg
ip route add 10.10.10.103 dev ggg

vim /usr/local/bin/macvlan_stop.sh

ip route del 10.10.10.100 dev ggg || true
ip route del 10.10.10.101 dev ggg || true
ip route del 10.10.10.102 dev ggg || true
ip route del 10.10.10.103 dev ggg || true
ip link set ggg down || true
ip addr del 10.10.10.99 dev ggg || true
ip link del ggg || true

保存，开机启动
